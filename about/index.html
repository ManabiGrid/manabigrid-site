ABOUT THIS EXHIBITION

GitHubを知らなくても、ここで学べます。

このサイトは、GitHubに保存されたManabiGridの教材原稿を、読みやすい形へ自動変換した展示版です。教材を読むだけなら、GitHubのアカウントも操作も必要ありません。

01 / READ

このサイトの仕事

教科、単元、レッスンの順に開き、Markdownの記号を意識せずに教材を読むための場所です。登録、学習履歴の保存、アクセス解析はありません。検索語も端末の外へ送りません。

教材をさがす

02 / REPOSITORY

リポジトリページは、この順番で見る

  1. まずREADMEを読む。プロジェクトの目的、いまある教材、注意点がまとまっています。
  2. フォルダ名を選んで、下へたどる。教材本文はmaterials/、進捗と単元一覧はcurriculum/にあります。
  3. 教材ファイルを開く。lesson_01.mdのようなMarkdownが原稿、assets/のSVGが図版です。

GitHubでリポジトリを見る（外部サイト・新しいタブで開きます）

03 / NO ACCOUNT

アカウントがなくても、閲覧と保存ができる

ページを見るだけならサインイン不要です。教材一式を保存したい場合は、リポジトリ上部のCodeを開き、Download ZIPを選びます。ZIPはその時点のファイル一式で、自動更新はされません。

04 / ISSUE

Issueで誤りを知らせる3ステップ

  1. GitHubアカウントを作る（外部サイト・新しいタブで開きます）か、持っているアカウントでサインインします。
  2. リポジトリのIssuesを開き、New issueを選びます。
  3. 誤り報告テンプレートを選び、対象ファイルと疑問点を書きます。

Issueテンプレートを選ぶ（外部サイト・新しいタブで開きます） ／ 誤り報告を直接開く（外部サイト・新しいタブで開きます）

大切: Issueはインターネット全体に公開されます。名前、学校名、住所、連絡先、顔写真などは書かないでください。アカウントを作りたくない場合は、おたよりフォーム（外部サイト・新しいタブで開きます）を使えます。フォームはManabiGrid運営へ名前やメールアドレスを渡さず送れますが、Google Forms側のデータ取扱いにはGoogleの規約が適用されます。

05 / WORDS

GitHub用語のミニ辞典

リポジトリ
教材や図版、説明文をひとまとめに保管する場所。
コミット
「この時点の変更」を保存した記録。展示版は特定のコミットから作られます。
Issue
誤りや提案を、公開の話題として記録する場所。
Markdown
見出しや箇条書きを記号で表す、原稿向けのテキスト形式。

この展示は正本の更新を定期的に確認し、変化があったときだけ再生成する設計です。リンク、本文、図版、来歴、公開検疫のすべてに通った生成物だけを次の展示候補にします。失敗時は直前の公開版を残します。
